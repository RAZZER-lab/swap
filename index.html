Privacy-First Design

Built on Monero's proven cryptography, Zephyr ensures untraceable transactions through ring signatures, stealth addresses, and confidential transactions.

Multi-Asset System

Four native assets working in harmony: ZPH (base coin), ZSD (stablecoin), ZRS (reserve shares), and ZYS (yield shares).

Djed Protocol

Based on the Minimal Djed protocol, ensuring ZSD stablecoin is over-collateralized by ZPH held in the reserve pool.

Key RPC Method: transfer

The transfer method handles sending funds and converting between Zephyr's native assets. Here's an example of sending 1 ZEPH: 

curl http://127.0.0.1:17777/json_rpc -d '{
  "jsonrpc": "2.0",
  "id": "0",
  "method": "transfer",
  "params": {
    "destinations": [
      {
        "address": "ZEPHXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX",
        "amount": 1000000000000
      }
    ],
    "source_asset": "ZPH",
    "destination_asset": "ZPH",
    "priority": 1,
    "mixin": 16,
    "ring_size": 17,
    "unlock_time": 0,
    "get_tx_key": true
  }
}' -H 'Content-Type: application/json'

Native Assets Architecture

Zephyr (ZPH)

Base Coin

The main cryptocurrency of the Zephyr network. Used for transaction fees, mining rewards, and as primary collateral.

Asset Type: ZPH
Ticker: $ZEPH

Zephyr Stable Dollar (ZSD)

Stablecoin

Native stablecoin pegged to the US Dollar. Minted by locking ZPH as collateral and can be redeemed for ZPH.

Asset Type: ZSD
Ticker: $ZSD

Zephyr Reserve Share (ZRS)

Reserve Pool Share

Shares in the Zephyr reserve pool that backs the ZSD stablecoin. Holders benefit from fees and reserve equity.

Asset Type: ZRS
Ticker: $ZRS

Zephyr Yield Share (ZYS)

Yield Instrument

Represents a user's share in the ZSD Yield Reserve. Generated by staking ZSD and entitles holders to yield rewards.

Asset Type: ZYS
Ticker: $ZYS

Minting Process

Users lock ZPH as collateral to mint ZSD (stablecoin) or ZRS (reserve shares) according to the protocol's collateralization requirements.

Redemption Process

ZSD and ZRS can be redeemed for ZPH at the current redemption rate, maintaining the stablecoin's peg through algorithmic mechanisms.

Critical Integration Notes

Asset Type Requirements

All RPC calls involving transactions must explicitly specify asset types. Failure to check asset_type fields will result in incorrect balance calculations.

// Always check asset_type
if (tx.asset_type === "ZPH") {
// Handle ZEPH transaction
} else if (tx.asset_type === "ZSD") {
// Handle ZSD transaction
}

Deprecated Asset Types

Old asset types (ZEPH, ZEPHUSD, ZEPHRSV, ZYIELD) are no longer valid. Only the new audited asset types (ZPH, ZSD, ZRS, ZYS) should be used.

Important: MEXC exchange balances were specifically handled in Hardfork v11. Other exchanges must update their systems to recognize new asset types.
